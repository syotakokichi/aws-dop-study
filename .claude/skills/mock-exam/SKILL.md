# 模擬試験問題生成スキル

DOP-C02 形式の模擬試験問題を生成します。

## 使い方

```
/mock-exam [ドメイン番号または "all"] [問題数]
```

### 例

```
/mock-exam 1 5          # ドメイン1から5問生成
/mock-exam all 10       # 全ドメインから10問生成
/mock-exam 2 3          # ドメイン2から3問生成
```

## 動作

1. 指定されたドメインのサービス学習ファイルを読み込む
2. 試験頻出ポイントと出題パターンを参照
3. シナリオベースの問題を生成
4. exam-prep/ ディレクトリに保存
5. 回答後に正答率を PROGRESS.md に記録

## 問題形式

DOP-C02 の実際の試験形式に準拠:

- **シナリオベース**: 実際の業務シナリオを提示
- **4択または複数選択**: A〜Dの選択肢
- **ベストプラクティス重視**: AWSの推奨アプローチを問う
- **トレードオフ考慮**: コスト、パフォーマンス、セキュリティのバランス

## 出力例

```markdown
## 問題 1 (ドメイン1: SDLC自動化)

### シナリオ
あなたの会社では、複数のAWSアカウント（開発、ステージング、本番）を
使用しています。CodePipelineを使用して、開発アカウントのCodeCommit
リポジトリから本番アカウントへの自動デプロイを実現したいと考えています。

### 質問
この要件を満たすために必要な設定として、最も適切な組み合わせはどれですか？

### 選択肢
A. 本番アカウントでIAMロールを作成し、開発アカウントのCodePipelineサービスロールにAssumeRole権限を付与する
B. 開発アカウントのCodePipelineに本番アカウントの管理者認証情報を保存する
C. 両アカウントでCodePipelineを別々に作成し、S3で同期する
D. AWS Organizationsを使用して、すべてのアカウントを単一のアカウントに統合する

### 正解
A

### 解説
クロスアカウントデプロイでは、ターゲットアカウントにIAMロールを作成し、
ソースアカウントのサービスロールがそのロールを引き受ける（AssumeRole）
設定が必要です。また、KMSキーとS3バケットのポリシーも適切に設定する
必要があります。

### 関連トピック
- codepipeline.md: クロスアカウントデプロイの設定
- 試験頻出ポイント: クロスアカウントデプロイの設定（IAMロール、KMS）
```

| 3. 監視・ロギング | 15% | 2問 |
| 4. ポリシー・標準 | 10% | 1問 |
| 5. インシデント対応 | 18% | 2問 |
| 6. 高可用性・耐障害性 | 18% | 1問 |
