# 学習進捗記録スキル

学習進捗を PROGRESS.md に記録します。

## 使い方

```
/log-progress [ドメイン番号] [サービス名] [学習時間] [学習内容]
```

### 例

```
/log-progress 1 CodePipeline 2h "パイプライン構造、クロスアカウントデプロイ"
/log-progress 2 CloudFormation 1.5h "ネストスタック、変更セット"
/log-progress 6 Auto Scaling 1h "ターゲット追跡ポリシー"
```

## 動作

1. PROGRESS.md を読み込む
2. 指定されたドメインの学習項目テーブルを更新
3. 理論/ハンズオンの進捗を ⬜ → 🔄 または ✅ に更新
4. 本日の日付を記録
5. 週次目標のチェックボックスを更新（該当する場合）

## ドメイン対応表

| 番号 | ドメイン名 | 配点 |
|------|-----------|------|
| 1 | SDLC自動化 | 22% |
| 2 | 構成管理・IaC | 17% |
| 3 | 監視・ロギング | 15% |
| 4 | ポリシー・標準 | 10% |
| 5 | インシデント・イベント対応 | 18% |
| 6 | 高可用性・耐障害性 | 18% |

## 進捗マーク

- ⬜ 未着手
- 🔄 進行中
- ✅ 完了

## 注意事項

- 理論学習とハンズオンは別々に記録することを推奨
- 学習時間は累積で管理
- 模擬試験の正答率は別途 `/mock-exam` で記録
