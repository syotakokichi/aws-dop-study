# 試験直前チートシート

## デプロイ戦略比較

| 戦略 | ダウンタイム | ロールバック | リスク | ユースケース |
|------|-------------|--------------|--------|--------------|
| In-Place | あり | 遅い | 高 | 開発環境 |
| Rolling | 最小 | 遅い | 中 | コスト重視 |
| Blue/Green | なし | 即時 | 低 | 本番環境 |
| Canary | なし | 即時 | 最低 | 重要サービス |

## CloudFormation スタック操作

| 操作 | 説明 |
|------|------|
| CREATE_IN_PROGRESS | 作成中 |
| CREATE_COMPLETE | 作成完了 |
| CREATE_FAILED | 作成失敗 |
| UPDATE_IN_PROGRESS | 更新中 |
| UPDATE_COMPLETE | 更新完了 |
| UPDATE_ROLLBACK_COMPLETE | 更新失敗→ロールバック完了 |
| DELETE_IN_PROGRESS | 削除中 |

## CloudWatch メトリクス解像度

| タイプ | 解像度 | 保持期間 |
|--------|--------|----------|
| 標準 | 1分 | 15日（1分）→ 63日（5分）→ 15ヶ月（1時間） |
| 高解像度 | 1秒 | 3時間（1秒）→ 以降は集約 |

## DR戦略（RTO順）

| 戦略 | RTO | RPO | コスト |
|------|-----|-----|--------|
| Backup & Restore | 時間〜日 | 最大24時間 | 最低 |
| Pilot Light | 10分〜時間 | 分 | 低 |
| Warm Standby | 分 | 秒 | 中 |
| Active-Active | リアルタイム | ゼロ | 最高 |

## Auto Scaling ポリシー

| タイプ | 説明 | ユースケース |
|--------|------|--------------|
| ターゲット追跡 | 目標値を維持 | CPU 50%維持など |
| ステップ | 段階的スケーリング | 細かい制御 |
| シンプル | 1アクション | 単純なケース |
| 予測 | ML予測でプロビジョニング | パターンのあるワークロード |

## Route 53 ルーティングポリシー

| ポリシー | ユースケース |
|----------|--------------|
| シンプル | 単一リソース |
| 加重 | A/Bテスト、段階的移行 |
| レイテンシ | 最速リージョンへ |
| フェイルオーバー | DR |
| 地理的位置 | コンテンツローカライゼーション |
| 地理的近接性 | トラフィックシフト |
| マルチバリュー | シンプルな負荷分散 |

## ELB比較

| タイプ | レイヤー | プロトコル | ユースケース |
|--------|---------|------------|--------------|
| ALB | L7 | HTTP/HTTPS | Webアプリ |
| NLB | L4 | TCP/UDP/TLS | 低レイテンシ、高スループット |
| GWLB | L3 | IP | サードパーティアプライアンス |

## Systems Manager 主要機能

| 機能 | 説明 |
|------|------|
| Parameter Store | 設定・シークレット管理 |
| Secrets Manager | ローテーション付きシークレット |
| Session Manager | SSH代替（ポート不要） |
| Automation | ランブック実行 |
| Patch Manager | パッチ適用 |
| State Manager | 構成維持 |

## EventBridge vs SNS vs SQS

| サービス | パターン | 特徴 |
|----------|----------|------|
| EventBridge | イベント駆動 | パターンマッチング、スキーマ |
| SNS | Pub/Sub | ファンアウト、プッシュ |
| SQS | キュー | デカップリング、バッファリング |

## 覚えるべき数値

- SQS メッセージ保持: 最大14日（デフォルト4日）
- SQS 可視性タイムアウト: 最大12時間（デフォルト30秒）
- Lambda 同時実行: デフォルト1000
- Lambda タイムアウト: 最大15分
- Step Functions 履歴: 90日
- CloudWatch Logs 保持: 無期限可能
- Auto Scaling クールダウン: デフォルト300秒
